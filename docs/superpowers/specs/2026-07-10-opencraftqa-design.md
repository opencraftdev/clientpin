# OpenCraftQA — Design / Spec

**Date:** 2026-07-10
**Status:** Approved for planning
**Supersedes:** the private admin-panel model in `2026-07-09-qa-tagging-system-design.md`

## 1. Summary

A pivot from a private admin panel to **public, shareable, expiring QA lists**.

- A client uses the **Chrome extension** to pick or create a **project**, tag components on their live site (highlight → inline chat-bubble comment), and each tag captures a **screenshot** of the component.
- The extension generates a public link **`opencraftqa.com/<slug>`**. Anyone with the link sees that one list (screenshot + locate link + comment + status), can change statuses, and can copy **AI-fix prompts** (per item or for the whole list).
- **No accounts anywhere.** The extension remembers your projects locally. Lists are read by `slug`; the extension writes by `project_key`.
- Lists are **deleted 7 days after their last activity**.
- Web app deploys to **Vercel**; DB and Storage stay **Supabase**.

## 2. Goals & non-goals

**Goals (v1)**
- Extension: project dropdown + create; tag with inline bubble comment; screenshot capture; "View list" opens the slug URL.
- Public `/[slug]` list: screenshot thumbnail, locate link, comment, editable status, per-item and bulk AI-fix prompt copy.
- 7-day inactivity expiry with cleanup; expiry shown on the list.

**Non-goals (v1)** — fit the schema for later, do not build now:
- Accounts / login / ownership.
- Real LLM calls (AI-fix only copies a prompt).
- Reply threads, assignment, notifications.
- Editing/deleting tags from the web list (status change only).
- Server-side project listing (extension holds your projects locally).

## 3. Architecture

```
Chrome Extension (MV3)          Supabase (Postgres + Storage)      opencraftqa.com (Next → Vercel)
 popup = project dropdown        projects: slug, project_key,       public route /[slug]
   (chrome.storage)                last_active_at                    reads list via get_list(slug)
 create project → RPC            tags: anchor, comment, status,      screenshot + Locate + AI-fix
 tag → highlight → bubble          page_url, screenshot_path         no auth, no sidebar
 screenshot → Storage → path     RPCs (anon, SECURITY DEFINER):
 "View list" → slug URL            create_project / create_tag /
 background SW: captureVisibleTab  get_list / set_status
                                  Storage bucket `screenshots`
                                  pg_cron: purge >7d inactive
```

Access stays **RPC-only** (tables never granted to `anon`), consistent with today. Two ids: **`slug`** is the public view id; **`project_key`** is the write credential held only by the extension.

## 4. Data model

```
projects
  id             uuid pk default gen_random_uuid()
  name           text not null
  site_url       text not null
  slug           text unique not null   -- public share id (url-safe, ~10 chars)
  project_key    text unique not null   -- extension write credential
  created_at     timestamptz not null default now()
  last_active_at timestamptz not null default now()

tags
  id              uuid pk default gen_random_uuid()
  project_id      uuid not null references projects(id) on delete cascade
  page_url        text not null
  anchor          jsonb not null        -- { selector, text, nthOfType, tagName }
  comment         text not null
  status          text not null default 'new'
                    check (status in ('new','in_progress','resolved'))
  screenshot_path text                  -- Storage key, nullable
  created_at      timestamptz not null default now()
```

Changes from the previous model: `projects` **drops `owner`**, **adds `slug` + `last_active_at`**; `tags` **adds `screenshot_path`**. `slug` and `project_key` generated by `create_project` (e.g. `encode(gen_random_bytes(...), 'base64url')` or a nanoid-style function).

**RLS/grants:** RLS enabled, **no policies** on the tables (direct access denied); all access flows through the four SECURITY DEFINER RPCs granted to `anon`. Same posture as the current build.

## 5. RPCs (all `anon`, `security definer`, `set search_path = public`)

- `create_project(p_name text, p_site_url text) returns table(slug text, project_key text)` — inserts a project with generated `slug`/`project_key`, returns both.
- `create_tag(p_project_key text, p_anchor jsonb, p_comment text, p_page_url text, p_screenshot_path text) returns uuid` — validates key, inserts tag, bumps parent `last_active_at`. `p_screenshot_path` nullable.
- `get_list(p_slug text) returns table(...)` — returns the project (name, site_url, slug, last_active_at) and its tags. Used by the public web page. Returns nothing for unknown/expired slug.
- `get_tags(p_project_key text, p_page_url text) returns setof tags` — retained for the **extension** to draw pins for the active project on the current page.
- `get_tag(p_tag_id uuid) returns table(anchor jsonb, page_url text)` — used by the **locate** flow to resolve a single tag's anchor on the live page.
- `set_status(p_tag_id uuid, p_status text) returns void` — validates status, updates the tag, bumps parent `last_active_at`. Public (the list is a shared board).

## 6. Storage (screenshots)

- Bucket `screenshots`, **public read**, **anon insert** (policy allows anon uploads; no update/delete by anon).
- Path convention: `<slug>/<random-uuid>.png`. The extension holds the `slug` and mints a random filename client-side (the tag id does not exist yet at upload time, so it is not used in the path).
- The extension uploads the cropped PNG first, then passes the resulting path to `create_tag`.
- Public URL derived from the bucket path for display on the list.

## 7. Expiry

- A daily **pg_cron** job deletes `projects` where `last_active_at < now() - interval '7 days'`. Cascade removes tags.
- The same job removes orphaned Storage objects for deleted projects (delete objects under `<project_id>/`).
- The list page shows **"expires in N days"** computed from `last_active_at + 7 days`.

## 8. Chrome extension

**Popup** (`chrome.storage.local` holds `projects: [{ name, slug, project_key }]` and `activeSlug`):
- **Dropdown** of remembered projects; selecting one sets the active project.
- **New project**: name + site URL → `create_project` RPC → append `{name, slug, project_key}` to storage, set active.
- **View list** button → opens `https://opencraftqa.com/<activeSlug>` in a new tab.
- If no active project, tagging is disabled with a hint to pick/create one.

**Tag flow** (content script):
- Highlight + dim as today. On click of the target element:
  1. Show an **inline chat-bubble comment box** anchored near the element (textarea + Send/Cancel), replacing `prompt()`.
  2. On Send: request a **screenshot** from the background service worker (`chrome.tabs.captureVisibleTab`), **crop** to the element's rect × `devicePixelRatio` via an offscreen canvas, **upload** to Storage at `<slug>/<random-uuid>.png`.
  3. Call `create_tag(project_key, anchor, comment, page_url, screenshot_path)`.
  4. On success, drop a pin (as today). Screenshot is **best-effort**: on capture/upload failure, still create the tag with `screenshot_path = null`.
- **Pin-drawing** on the current page uses `get_tags(project_key, page_url)` for the active project (unchanged from today).
- **Locate on load**: if the page URL has `#qa-locate=<tagId>`, the content script calls `get_tag(tagId)`, re-finds the element via its anchor, scrolls to it and flashes the highlight.

**Manifest additions:** `tabs` permission; a background **service worker** for `captureVisibleTab`. Extension `.env` points at the hosted Supabase for production builds (local for dev).

## 9. Web app (`opencraftqa.com`, the repurposed `panel`)

**Remove:** `app/login`, auth `proxy.ts`/middleware, the `(app)` route group, the sidebar shell, project create/list pages, `createProject`/`signOut`/auth server actions, the Supabase server client's cookie/session wiring (no auth).

**Add:**
- Public route **`/[slug]/page.tsx`** — calls `get_list(slug)` (server, anon). Unknown/expired → `notFound()` ("This list was not found or has expired.").
- Header: project name, `site_url` link, **"expires in N days"**, and a **Fix all** button (copies `buildBulkPrompt(openTags)`).
- Each tag row/card: **screenshot thumbnail** (from Storage public URL; placeholder if null), **Locate** link (`<page_url>#qa-locate=<tagId>` in a new tab), comment, **status select** (calls `set_status` — public), **AI Fix** button (copies `buildPrompt(tag)`).
- Root `/` — a minimal landing (what the tool is + "tag via the extension"). No dashboard.

**AI-fix prompt (shared, pure, unit-tested):**
```
buildPrompt(tag):
  Fix this UI issue on <page_url>:

  Problem: "<comment>"
  Element: <<tagName>>  (selector: <anchor.selector>)
  Status: <status>

  Apply a minimal, accessible fix consistent with the existing design system.

buildBulkPrompt(tags):  # header + numbered list of the above bodies, open items only
```

## 10. Error handling

- Unknown/expired `slug` → `notFound()` with a friendly message.
- `create_project` / `create_tag` errors → surfaced in the extension (inline, not a crash).
- Screenshot capture or upload failure → tag still saves (`screenshot_path` null); the row shows a "no screenshot" placeholder.
- Locate: element not found on the live page → no scroll, screenshot still conveys the target.
- `set_status` on a since-expired list → RPC no-ops/errors; the page shows the not-found state on reload.

## 11. Testing

- **Postgres:** `create_project` returns a unique slug + key; `get_list` returns a project's tags by slug and nothing for a bad slug; `create_tag` and `set_status` bump `last_active_at`; the expiry query selects only >7-day-inactive projects; RLS denies direct table access to `anon`.
- **Extension:** existing anchor round-trip (unchanged); screenshot **crop math** (rect × dpr → canvas coords) as a pure function; `buildPrompt` if colocated.
- **Web:** `buildPrompt` / `buildBulkPrompt` pure-function tests (exact output); `/[slug]` renders a list from a stubbed `get_list` and 404s on unknown slug.

## 12. Build order

1. **DB** — migration (slug, last_active_at, screenshot_path, drop owner), the four RPCs, Storage bucket + policies, pg_cron expiry. Usable via `psql`/seed before any client.
2. **Web app** — strip auth/sidebar; public `/[slug]` list with screenshot, locate link, status, AI-fix (per-item + bulk). Exercisable with seeded data before the extension changes.
3. **Extension** — project dropdown + create; inline bubble comment; background screenshot capture + Storage upload; `create_tag` with screenshot; View-list URL; locate-on-load.

Each layer is independently testable before the next.
